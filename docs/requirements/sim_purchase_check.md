# Сценарий использования: SIM‑1 — Проверка покупки

**Вопрос пользователя:**  
«Хватит ли денег на покупку? На сколько дней хватит до зарплаты?»

---

## Исходные данные (ввод пользователя)

| Параметр | Способ получения | Пример |
|----------|------------------|--------|
| Сумма покупки | Пользователь вводит число | 12 000 ₽ |
| Количество дней до зарплаты | Пользователь вводит целое число | 15 дней |
| Ежемесячные расходы (все, кроме подписок) | Пользователь вводит сумму | 45 000 ₽ |

> Примечание: среднедневные расходы вычисляются автоматически как `ежемесячные_расходы / 30`.

---

## Алгоритм расчёта

1. **Проверка достаточности баланса**  
   - Баланс `B` берётся из БД (текущие доходы − расходы).  
   - Если `B < сумма_покупки` → покупка невозможна, сообщить дефицит.

2. **Остаток после покупки**  
   `R = B - сумма_покупки`

3. **Среднедневные расходы**  
   `daily_avg = ежемесячные_расходы / 30`

4. **Дней хватит остатка**  
   `days_covered = floor(R / daily_avg)`, где `floor` — округление вниз.

5. **Сравнение с днями до зарплаты**  
   - Если `days_covered >= days_to_salary` → «Хватит до зарплаты и ещё на X дней».  
   - Если `0 < days_covered < days_to_salary` → «Хватит только на N дней, до зарплаты не хватит на M дней».  
   - Если `days_covered <= 0` → «После покупки денег не останется».

---

## Результат

Пользователь получает текстовый прогноз с рекомендацией.

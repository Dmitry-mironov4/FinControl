# Сценарий использования: SIM‑4 — Сокращение расходов в категории

**Вопрос пользователя:**  
«Если я сокращу расходы на X% в категории, когда я достигну цели?»

---

## Исходные данные

| Параметр | Источник |
|----------|----------|
| Цель (желаемая сумма, накопления, ежемесячный взнос) | Из БД (цель пользователя) |
| Общие ежемесячные расходы | Из БД (сумма всех расходов за последние 30 дней) |
| Категория для сокращения | Пользователь выбирает из выпадающего списка (Dropdown) |
| Процент сокращения | Пользователь вводит число (0–100) |

> **category_share** (доля категории в расходах) вычисляется автоматически на основе истории трат за последние 3 месяца. Если истории недостаточно (менее 2 месяцев), то по умолчанию режутся **все расходы целиком** (т.е. `category_share = 1`).

---

## Алгоритм

1. **Расходы на выбранную категорию**  
   `category_expense = total_expenses * category_share`

2. **Экономия в месяц**  
   `savings = category_expense * (cut_percent / 100)`

3. **Новый ежемесячный взнос в цель**  
   `new_monthly_savings = current_monthly_savings + savings`

4. **Новый срок достижения цели**  
   `remaining = target - current_savings`  
   `new_months = ceil(remaining / new_monthly_savings)`

5. **Ускорение**  
   `months_saved = current_months - new_months`

---

## Результат

Сообщение: «Сократив расходы в категории X на Y%, вы будете откладывать на Z ₽ больше и достигнете цели на N месяцев раньше.»  
Если данных о категории нет — «Вы не можете сократить именно эту категорию, будут сокращены все расходы равномерно.»

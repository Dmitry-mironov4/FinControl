# Сценарий использования: «Хватит ли денег на покупку? На сколько дней хватит до зарплаты?»  

---

## 1. Назначение сценария  
Пользователь хочет принять решение о покупке. Он вводит сумму предполагаемой траты. Приложение на основе текущих данных о финансах пользователя вычисляет:

- Достаточно ли средств на счёте для этой покупки.
- Если да — на сколько дней хватит оставшихся денег с учётом ежедневных расходов до момента получения зарплаты (или иного регулярного дохода).

Результат помогает пользователю понять, не останется ли он без денег до следующего поступления.

---

## 2. Исходные данные (хранятся в системе или вводятся пользователем)

| Параметр | Обозначение | Источник | Пример |
|----------|-------------|----------|--------|
| Текущий баланс | `B` | Рассчитывается из доходов и расходов | 15 000 ₽ |
| Сумма покупки (ввод пользователя) | `P` | Поле ввода | 12 000 ₽ |
| Дата следующей зарплаты | `Date_salary` | Из регулярного дохода (пользователь указал) | 20.04.2026 |
| Среднедневные траты | `Day_expences_avg` | Автоматически за последние 30 дней | 1 200 ₽/день |

> *Примечание:* если история трат отсутствует, приложение запрашивает у пользователя примерный дневной бюджет.

---

## 3. Пошаговый алгоритм расчёта (с формулами)

### Шаг 1. Проверка достаточности баланса

- Если B ≥ P:
→ Покупка возможна.
→ Остаток после покупки: R = B - P
- Иначе:
→ Покупка невозможна.
→ Дефицит: D = P - B
→ Сообщение пользователю: «Не хватает D ₽. Покупка отменяется».
→ Сценарий завершается.


### Шаг 2. Расчёт количества дней до зарплаты

* Сегодняшняя дата: Date_today
* Дата зарплаты: Date_salary
* Дней до зарплаты: N_salary = (Date_salary - Date_today).days


Если `N_salary ≤ 0` → пользователь должен указать следующую дату зарплаты (периодичность).

### Шаг 3. Расчёт, на сколько дней хватит остатка

При условии, что среднедневные траты Day_expences_avg > 0:
N_cover = floor(R / D_avg)

`floor` — округление вниз до целого числа дней.

### Шаг 4. Сравнение и вывод результата

| Условие | Результат для пользователя |
|---------|----------------------------|
| `N_cover ≥ N_salary` | «Денег хватит до зарплаты и ещё на X дней сверх.» <br> X = N_cover − N_salary |
| `0 < N_cover < N_salary` | «После покупки денег хватит только на N_cover дней. До зарплаты останется ещё N_salary − N_cover дней без денег. Рекомендуем отложить покупку или снизить ежедневные траты.» |
| `N_cover = 0` | «После покупки деньги закончатся сразу. До зарплаты придётся обходиться без трат.» |

---

## 5. Исключительные ситуации (как должно вести себя приложение)

| Ситуация | Действие приложения |
|----------|---------------------|
| Не задана дата зарплаты | Запросить пользователя: «Укажите дату следующего поступления денег» |
| Среднедневные траты = 0 (нет истории) | Использовать значение по умолчанию (например, 500 ₽) или попросить пользователя ввести дневной бюджет |
| Остаток после покупки = 0 | Сообщить: «После покупки денег не останется. До зарплаты вы сможете продержаться, только если вообще ничего не тратить.» |
| Покупка превышает баланс | Красное сообщение с указанием суммы недостающих средств |

# Сценарий использования: «Если я куплю это, на сколько месяцев отодвинется моя цель?»  

---

## 1. Назначение сценария  
У пользователя есть финансовая цель (например, накопить на отпуск, квартиру, машину). Он рассматривает возможность совершить покупку, которая повлияет на его накопления. Приложение рассчитывает, на сколько месяцев увеличится срок достижения цели, если он купит этот товар сейчас.

Результат помогает пользователю осознать, стоит ли тратить деньги на текущую покупку в ущерб главной цели.

---

## 2. Исходные данные (хранятся в системе или вводятся пользователем)

| Параметр | Обозначение | Источник | Пример |
|----------|-------------|----------|--------|
| Цель: желаемая сумма | `G` | Задаётся пользователем при создании цели | 100 000 ₽ |
| Текущие накопления (под цель) | `S` | Вносится пользователем или вычитается из общего баланса | 40 000 ₽ |
| Ежемесячный платёж / сумма откладывания | `M` | Задаётся пользователем (сколько готов откладывать в месяц) | 10 000 ₽/мес |
| Сумма покупки (ввод пользователя) | `P` | Поле ввода | 15 000 ₽ |

> *Примечание:* предполагается, что покупка оплачивается из накоплений (либо уменьшает остаток на цели, либо увеличивает требуемую сумму цели — в зависимости от логики).

---

## 3. Пошаговый алгоритм расчёта (с формулами)

### Шаг 1. Проверка наличия цели
Если у пользователя нет активной цели:
→ Сообщение: «Сначала создайте финансовую цель (например, накопить на отпуск)».
→ Сценарий завершается.



### Шаг 2. Расчёт текущего срока достижения цели (без покупки)
Остаток до цели: R_current = G - S
Если R_current ≤ 0:
→ Цель уже достигнута! Покупка не отодвинет срок (или можно показать 0 месяцев).
→ Сценарий завершается.
Если M > 0:
T_current = ceil(R_current / M) // в месяцах, округление вверх

`ceil` — округление до целого вверх (например, 2.3 месяца → 3 месяца).

### Шаг 3. Расчёт новых накоплений после покупки
Покупка оплачивается из накоплений, поэтому:
S_new = S - P


Если `S_new < 0` → покупка невозможна (недостаточно накоплений).  
В этом случае сообщаем: «Недостаточно накоплений для покупки. Нужно ещё X ₽».

### Шаг 4. Расчёт нового срока достижения цели после покупки
Остаток до цели после покупки: R_new = G - S_new (или G - (S - P) = R_current + P)
R_new = R_current + P


Если `M > 0`:
T_new = ceil(R_new / M)


### Шаг 5. Расчёт сдвига срока
ΔT = T_new - T_current // в месяцах


- Если `ΔT = 0` → «Цель отодвинется менее чем на месяц (или не отодвинется)».
- Если `ΔT > 0` → «Покупка отодвинет достижение цели на ΔT месяцев».

### Шаг 6. Вывод результата пользователю

| Ситуация | Результат |
|----------|-----------|
| `S_new < 0` | «Недостаточно накоплений. Нужно ещё |S_new| ₽.» |
| `ΔT = 0` | «Цель не отодвинется (или отодвинется менее чем на месяц).» |
| `ΔT = 1` | «Цель отодвинется на 1 месяц.» |
| `ΔT > 1` | «Цель отодвинется на ΔT месяцев.» |

**Дополнительно** можно показать:
- Текущий срок: `T_current` месяцев.
- Новый срок: `T_new` месяцев.

---

## 5. Исключительные ситуации (как должно вести себя приложение)

| Ситуация | Действие приложения |
|----------|---------------------|
| Нет активной цели | Предложить создать цель («Сначала задайте цель накопления») |
| Цель уже достигнута | Сообщить: «Цель уже достигнута. Покупка не повлияет на срок.» |
| Ежемесячный платёж `M = 0` (пользователь не откладывает) | Попросить указать, сколько он может откладывать в месяц, иначе расчёт невозможен |
| Сумма покупки `P` больше текущих накоплений `S` | Красное сообщение: «Не хватает X ₽ на покупку. Отложите покупку или пополните накопления.» |
| После покупки `S_new = 0` | «Накопления обнулены. Цель начнётся с нуля, срок увеличится на T_new − T_current месяцев.» |
| Пользователь меняет ежемесячный платёж | Пересчитать динамически |

# Сценарий использования: «Если я сокращу расходы на X%, когда я достигну цели?»  


---

## 1. Назначение сценария  
Пользователь хочет понять, как уменьшение его регулярных расходов на определённый процент ускорит достижение финансовой цели. Приложение на основе текущей цели (сумма, накопления, ежемесячный платёж) и введённого процента сокращения расходов пересчитывает новый ежемесячный взнос в цель и показывает, на сколько месяцев уменьшится срок достижения цели.

Результат мотивирует пользователя оптимизировать бюджет и показывает выгоду от экономии.

---

## 2. Исходные данные (хранятся в системе или вводятся пользователем)

| Параметр | Обозначение | Источник | Пример |
|----------|-------------|----------|--------|
| Цель: желаемая сумма | `G` | Задаётся пользователем | 100 000 ₽ |
| Текущие накопления под цель | `S` | Из цели | 40 000 ₽ |
| Текущий ежемесячный платёж (сумма, которую пользователь откладывает на цель) | `M_current` | Из цели | 10 000 ₽ |
| Ежемесячные регулярные расходы (все, кроме взноса в цель) | `Expenses_current` | Из учёта расходов пользователя | 30 000 ₽ |
| Процент сокращения расходов (ввод пользователя) | `X` | Поле ввода (целое число от 1 до 100) | 20% |

> *Примечание:* предполагается, что сэкономленная сумма полностью направляется на увеличение ежемесячного взноса в цель.

---

## 3. Пошаговый алгоритм расчёта (с формулами)

### Шаг 1. Проверка наличия цели и корректности ввода
Если у пользователя нет активной цели:
→ Сообщение: «Сначала создайте финансовую цель».
→ Сценарий завершается.

Если X ≤ 0 или X > 100:
→ «Введите процент от 1 до 100».
→ Сценарий завершается.



### Шаг 2. Расчёт текущего срока достижения цели (без экономии)
Остаток до цели: R = G - S
Если R ≤ 0:
→ Цель уже достигнута. Сокращение расходов не требуется.
→ Сценарий завершается.
Если M_current > 0:
T_current = ceil(R / M_current) // в месяцах, округление вверх



### Шаг 3. Расчёт экономии в месяц

Category_expense = Expenses_current × category_share
Savings = Category_expense × (cut_pct / 100)

Эти деньги добавляются к ежемесячному взносу в цель.

### Шаг 4. Расчёт нового ежемесячного взноса
M_new = M_current + экономия



### Шаг 5. Расчёт нового срока достижения цели
Если M_new > 0:
T_new = ceil(R / M_new)
Иначе:
→ T_new = бесконечность (невозможно достичь)



### Шаг 6. Расчёт ускорения (на сколько месяцев раньше)
ΔT = T_current - T_new // положительное число — ускорение


### Шаг 7. Вывод результата пользователю

| Ситуация | Результат |
|----------|-----------|
| `M_new > 0` и `ΔT > 0` | «Сократив расходы на X%, вы будете откладывать на M_new ₽ в месяц (на экономия ₽ больше). Цель будет достигнута через T_new месяцев вместо T_current. Вы достигнете цели на ΔT месяцев раньше.» |
| `ΔT = 0` | «Сокращение расходов на X% не ускорит достижение цели (изменение менее месяца).» |
| `M_new == 0` (невозможно) | «Сокращение расходов не даёт дополнительных средств для откладывания. Проверьте данные.» |

**Дополнительно** можно показать сумму экономии в деньгах.

---

## 5. Исключительные ситуации (как должно вести себя приложение)

| Ситуация | Действие приложения |
|----------|---------------------|
| Нет активной цели | Предложить создать цель |
| Цель уже достигнута | Сообщить: «Цель уже достигнута. Сокращение расходов не требуется.» |
| `M_current = 0` (пользователь ничего не откладывает) | Попросить указать, сколько он может откладывать в месяц, иначе расчёт невозможен |
| `Expenses_current = 0` (нет расходов) | Сообщить: «У вас нет регулярных расходов, которые можно сократить.» |
| Процент сокращения приводит к нулевым расходам (X = 100) | Предупредить: «Сокращение всех расходов до нуля — маловероятно. Результат может быть нереалистичным.» |
